2. Forecast Engines

Four engines, each serving a different purpose. The three-way engine is the most complex.

13-Week Direct Method — transaction-level, highest near-term accuracy

Inputs: opening bank balance, outstanding invoices, outstanding bills, recurring streams, weekly opex baseline.

Per-week loop (w = 1 to 13):

1
Customer collections = ∑(invoice.amount where due_date ∈ this week). Week 1 also sweeps overdue invoices.
2
Supplier payments = ∑(bill.amount where due_date ∈ this week). Week 1 also sweeps overdue bills.
3
Recurring stream expenses: for each stream, if next_expected ∈ this week, add median_amount to its component bucket (rent, payroll, utilities…), advance next_expected by frequency.
4
Residual opex = max(weekly_opex − total_recurring_this_week, 0)
5
Net cash flow = total_receipts − total_disbursements
6
Closing balance = opening + net. Carry forward as next week’s opening.
Output: 13 × WeekProjection (receipts, disbursements by component, net, opening/closing balance) + ForecastMetrics (min cash, runway, averages).
5-Year Indirect Method — rate-based projection, 260 weeks

Inputs: base_weekly_revenue (from 13-week avg or metrics), growth rate, seasonal indices, recurring streams, inflation, fixed_ratio, DSO trend, invoice CV.

Pre-loop:

residual_weekly = max(avg_disbursements − total_recurring_weekly, 0)
blended_rate = (inflation + growth) / 2

Per-week loop (w = 1 to 260):

1
Receipts = base × (1+g)year × S(month). If DSO trend > 1 d/q: × max(0.70, 1 − drag × year)
2
Fixed stream costs: each × (1+inflation)year. Variable stream costs: each × (1+growth)year.
3
Residual costs = residual × (1+blended)year × dampened_seasonal, where dampened = 1 + (S−1)(1−f).
4
Net = receipts − disbursements. Closing = opening + net.
5
Uncertainty: σ(w) = min(BASE + k√(w−1), 0.40). Spread = |balance| × σ.
Output: 260 weekly + 60 monthly projections, each with uncertainty bounds.
Three-Way Engine — P&L → BS → CF, the core model

Inputs: all 15 assumptions, BS opening positions, classified accounts, seasonal indices, base revenue, driver overrides, event injections, P&L cache.

9 steps, executed in strict order:

0
Partition events into recurring (hire, asset purchase…) and punctual (issue invoice, dividend, equity injection…). Expand each into per-week EventEffect(week, pl_effects, bs_effects).
1
Project P&L (260 weeks). Per week, resolve driver overrides (latest-start-week wins), then:
revenue = base_monthly / 4.33 × (1+growth)year × S(month)
COGS = revenue × (1 − margin)
staff = named_opex / 4.33 × (1+inflation)year  (FIXED)
marketing = named_opex / 4.33 × (1+growth)year  (VARIABLE)
other_opex = residual / 4.33 × (1+blended)year × dampened_S
depreciation = dep_monthly / 4.33  (flat)
interest = loan_balance × (BoE + spread) / 52
corp_tax = max(0, PBT × rate)
net_profit = PBT − tax
2
Apply event P&L effects. Add recurring then punctual deltas to each P&L line. Recompute gross profit, opex, PBT, tax, net profit.
3
Compute punctual BS deltas. For each one-off revenue event: create a receivable, clear after DSO days. For each one-off cost: create a payable, clear after DPO days. Mini-ledger with collection lag = ceil(DSO/7) weeks.
4
Roll forward BS (260 weeks). Per week, resolve DSO/DPO/capex/dividend overrides, apply trends, then:
trade_receivables = revenue × effective_DSO / 7  (excluding punctual revenue)
trade_payables = COGS × effective_DPO / 7
PPE = opening + cumulative_capex − cumulative_depreciation
VAT: net = (revenue × 20%) − (COGS + cash_opex) × 20% — accumulates, drains quarterly
PAYE: staff × 25% — accumulates, drains every 4 weeks
Corp tax: from P&L — accumulates, drains annually
retained_earnings = prev + net_profit − dividends
cash = total_L_E − non_cash_assets  (THE PLUG)
5
Apply event BS effects. Cumulative stock changes: PPE, bank_loans, share_capital from event effects. Merge punctual receivables/payables from the mini-ledger.
6
Recompute cash plug after all event modifications. $A = L + E$ must hold every week.
7
Derive CF (indirect method, 260 weeks):
CFO = net_profit + depreciation − ΔAR + ΔAP + ΔVAT + Δcorp_tax + ΔPAYE + Δother_WC
CFI = −capex
CFF = new_borrowing + loan_repayments + dividends + equity_injections
ΔCash = CFO + CFI + CFF  — must equal BS_cash(w) − BS_cash(w−1)
8
Aggregate to monthly (up to 60 months). P&L and CF: sum weekly values. BS: snapshot last week of each month.
Output: 260 weekly + 60 monthly of each: P&L (18 fields), BS (21 fields), CF (21 fields), plus the assumptions used.
Smart Causal Forecast — macro signal overlay on 13-week baseline

Inputs: 13-week baseline forecast, business profile, current vs baseline external factors.

1
Determine relevant signals for this business (by industry, SIC codes, exposure flags).
2
Generate ImpactFragments for each signal where current ≠ baseline. Generators: employer NI, pension, base rate, living wage, FX, commodities. Each fragment = absolute or percentage delta on a cash flow component.
3
Apply fragments independently to baseline: result = baseline + ∑(absolute × intensity) + ∑(baseline × pct/100 × intensity). REPLACEMENT takes precedence. Cascade balances.
4
Calculate uncertainty from fragment count and certainty levels (heuristic strategy, quadrature combination).
Output: baseline + adjusted 13-week forecasts, uncertainty bounds, list of macro fragments with total impact.

Events are partitioned into recurring (ongoing weekly P&L effects) and punctual (one-off BS effects with DSO/DPO timing). Applied after P&L projection but before BS roll-forward. Cash plug recomputed after all event modifications.

Driver Override Resolution

9 overridable drivers: revenue_growth, gross_margin, inflation, DSO, DPO, corp_tax, capex, dividends, BoE_rate.

Each override: (driver, value, start_week, duration_weeks). At each forecast week, if multiple overrides match the same driver, latest start_week wins. If none match, base assumption value is used.

P&L engine resolves: growth, margin, inflation, BoE, corp_tax. BS engine resolves: DSO, DPO, capex, dividends.

Unified Two-Phase Forecast

Production mode runs two phases:

1
Phase 1: three-way engine for weeks 1–13. Uses actual invoices/bills for near-term accuracy where possible.
2
Phase 2: three-way engine for weeks 14–260, using Phase 1’s final BS as the opening position. Events shifted by −13 weeks.

Combined output: 260 weeks, seamless handover at week 13.
