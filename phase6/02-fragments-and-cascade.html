Fragment Application and Balance Cascade

Phase 6 — Scenarios and Sensitivity, Page 2

The problem

Two macro signals affect revenue simultaneously: sterling strengthens 3% (reducing export revenue) and inflation rises 2% (increasing nominal prices). If we apply these sequentially — multiply by 0.97 then by 1.02 — we get a different result depending on the order. Worse, after modifying any week's flows, all subsequent cash balances are wrong unless we recalculate them. We need order-independent fragment application and a systematic balance cascade.

1. Independent Fragment Application

A fragment is a single macro adjustment — an FX shift, an inflation change, a rate move. Each fragment has a percentage impact $p_i$ and an intensity $\iota_i$ (defaulting to 1). The key principle: each fragment computes its effect from the original baseline, not from the running total after prior fragments.

Theorem — Independent Fragment Application

Given baseline value $B$ and fragments with percentage impacts $p_1, p_2, \ldots, p_n$ and intensities $\iota_1, \iota_2, \ldots, \iota_n$, each fragment's delta is:

$$\delta_i = B \times \frac{p_i}{100} \times \iota_i$$

The adjusted value is:

$$\text{adjusted} = B + \sum_{i=1}^{n} \delta_i$$

This is order-independent: the result does not depend on which fragment is applied first.

Proof — Commutativity of Independent Application

Each $\delta_i$ depends only on $B$, $p_i$, and $\iota_i$ — none of which change when other fragments are applied. The adjusted value is $B + \sum_i \delta_i$. Since addition of real numbers is commutative and associative:

$$B + \delta_1 + \delta_2 = B + \delta_2 + \delta_1$$

for any permutation of the deltas. The result is identical regardless of the order in which fragments are evaluated.

More generally, for $n$ fragments and any permutation $\sigma$ of $\{1, \ldots, n\}$:

$$B + \sum_{i=1}^n \delta_{\sigma(i)} = B + \sum_{i=1}^n \delta_i$$

This follows directly from the commutativity of addition over the reals. No additional assumptions about the fragments are required — they need not be independent in any probabilistic sense, nor small in magnitude. The result is exact. ∎

2. Why Not Sequential?

Sequential application multiplies factors:

$$\text{Sequential (order 1)}: B \times (1 + p_1/100) \times (1 + p_2/100)$$
$$\text{Sequential (order 2)}: B \times (1 + p_2/100) \times (1 + p_1/100)$$

These two are equal (multiplication commutes), but both differ from independent application. With $B = \text{£}100{,}000$, $p_1 = -3$, $p_2 = +2$:

Sequential: $100{,}000 \times 0.97 \times 1.02 = 100{,}000 \times 0.9894 = \text{£}98{,}940$.

Independent: $\delta_1 = 100{,}000 \times (-0.03) = -3{,}000$. $\delta_2 = 100{,}000 \times 0.02 = +2{,}000$. Adjusted $= 100{,}000 - 3{,}000 + 2{,}000 = \text{£}99{,}000$.

The £60 difference arises because sequential application compounds: the +2% is applied to the already-reduced £97,000 rather than the original £100,000. With larger percentages or more fragments, this compounding effect grows. Independent application keeps each fragment's impact interpretable — "FX cost us £3k, inflation gained us £2k" — without cross-contamination.

The compounding error grows with scale

With three fragments — FX -5%, inflation +3%, volume +4% — the sequential result is:

$$B \times 0.95 \times 1.03 \times 1.04 = B \times 1.01764$$

The independent result is:

$$B + B(-0.05) + B(0.03) + B(0.04) = B(1 + 0.02) = B \times 1.02$$

For $B = \text{£}500{,}000$: sequential gives £508,820, independent gives £510,000. The £1,180 gap is the accumulated cross-terms. With 5 fragments at ±10% each, the gap can reach thousands of pounds per week — significant over a 260-week forecast.

More importantly, the independent method lets you produce an attribution table: "FX cost us £25,000, inflation added £15,000, volume added £20,000." The total is exactly the sum of the parts. Sequential application destroys this property — the "FX impact" depends on whether you applied inflation first.

3. Replacement Fragments

Some fragments are not percentage-based but replacement values (analogous to overrides from the previous page). A replacement fragment sets the value directly, ignoring all percentage fragments. When both types exist:

$$\text{adjusted} = \begin{cases} \text{replacement value} & \text{if a replacement fragment exists} \\ B + \sum_i \delta_i & \text{otherwise} \end{cases}$$

If multiple replacement fragments conflict, the same resolution rule applies: latest start_week wins.

Fragment priority hierarchy

When both percentage and replacement fragments target the same driver at the same week, the resolution is:

  1. If a replacement fragment exists, it takes absolute precedence. All percentage fragments are ignored for that driver/week combination.
  2. If no replacement exists, all percentage fragments are applied independently from the baseline.
  3. The result then passes through the adjustment algebra (Phase 6, p1): multipliers, deltas, and overrides from the scenario layer can further modify the fragment-adjusted value.

This creates a clean separation: fragments model macro conditions (FX, inflation, interest rates), whilst the adjustment algebra models business decisions (pricing changes, new contracts, cost cuts). The two layers compose without interference because fragments modify the baseline before the adjustment algebra runs.

4. The Balance Cascade

Modifying a single week's cash flows has consequences that ripple forward through every subsequent week's balance.

Theorem — Balance Cascade Invariant

For any week $w$ in the forecast, the following invariants must hold:

$$\text{opening}(w) = \text{closing}(w - 1)$$
$$\text{closing}(w) = \text{opening}(w) + \text{net}(w)$$

where $\text{net}(w) = \text{receipts}(w) - \text{disbursements}(w)$. These define a cumulative sum. If any week $k$'s net flow changes by $\Delta$, then $\text{closing}(w)$ changes by $\Delta$ for all $w \geq k$.

Proof — Cascade Propagation

Let $\text{net}'(k) = \text{net}(k) + \Delta$ for some modification at week $k$. Then:

$$\text{closing}'(k) = \text{opening}(k) + \text{net}'(k) = \text{closing}(k) + \Delta$$

For week $k+1$: $\text{opening}'(k+1) = \text{closing}'(k) = \text{closing}(k) + \Delta$. Since $\text{net}(k+1)$ is unchanged:

$$\text{closing}'(k+1) = \text{opening}'(k+1) + \text{net}(k+1) = \text{closing}(k+1) + \Delta$$

By induction, $\text{closing}'(w) = \text{closing}(w) + \Delta$ for all $w \geq k$. ∎

Algorithm: after modifying week $k$, recompute closing balances by iterating from $w = k$ to the last forecast week, applying the invariant at each step. The computational cost is $O(W - k)$ where $W$ is the total number of forecast weeks.

Cascade example

A 6-week forecast with opening cash £100,000 and the following net flows:

Week123456
Net flow£5,000£8,000-£3,000£12,000£6,000£4,000
Closing£105,000£113,000£110,000£122,000£128,000£132,000

Now suppose a scenario reduces week 3's net flow by £7,000 (from -£3,000 to -£10,000). The cascade:

Week123456
Net flow£5,000£8,000-£10,000£12,000£6,000£4,000
Closing£105,000£113,000£103,000£115,000£121,000£125,000

Every closing balance from week 3 onwards drops by exactly £7,000. Weeks 1-2 are unaffected. This is the cascade in action: a single modification propagates as a constant shift through all subsequent balances.

5. Macro Impact Formulas

Three macro variables that commonly generate fragments:

Foreign exchange

$$\text{FX impact} = \text{exposed\_revenue} \times \frac{\text{current\_rate} - \text{baseline\_rate}}{\text{baseline\_rate}} \times (-1)$$

The $(-1)$ factor reflects that GBP strengthening (rate increase for GBP/foreign) reduces foreign revenue when converted to sterling.

Interest rate

$$\text{Interest impact} = \text{debt\_service} \times 0.70 \times \left(\frac{\text{current\_rate}}{\text{baseline\_rate}} - 1\right)$$

The 0.70 factor estimates the interest portion of total debt service (the remainder being principal repayment).

Employer National Insurance

$$\text{NI impact} = \text{payroll} \times 0.60 \times \frac{\text{current\_rate} - \text{baseline\_rate}}{100}$$

The 0.60 factor estimates the NI-liable portion of total payroll (excluding pension contributions, benefits in kind, etc.).

Deriving the FX formula

Suppose a UK company earns €500,000/month from European clients. At the baseline rate of £1 = €1.15, this converts to £500,000 / 1.15 = £434,783. If sterling strengthens to £1 = €1.20:

$$\text{new GBP revenue} = \frac{500{,}000}{1.20} = \text{£}416{,}667$$

The impact is £416,667 - £434,783 = -£18,116. Using the formula:

$$\text{impact} = 434{,}783 \times \frac{1.20 - 1.15}{1.15} \times (-1) = 434{,}783 \times 0.04348 \times (-1) = -\text{£}18{,}904$$

The formula is an approximation (linear in the rate change), accurate for small movements. The exact impact requires dividing by the new rate, but the linear formula is sufficient for scenario analysis where the rate change is typically a few percent.

6. Worked Example

A 4-week revenue forecast for a company with 40% export exposure. Opening cash balance: £200,000.

Week1234
Revenue (baseline)£100,000£105,000£98,000£110,000
Costs (baseline)£80,000£82,000£79,000£85,000

Fragments: FX strengthens 5% ($p_1 = -5$, applied to 40% of revenue, so intensity $\iota_1 = 0.40$). Inflation +2% ($p_2 = +2$, intensity $\iota_2 = 1.0$ on revenue).

Week 1 revenue deltas:

$$\delta_{\text{FX}} = 100{,}000 \times \frac{-5}{100} \times 0.40 = -\text{£}2{,}000$$
$$\delta_{\text{inf}} = 100{,}000 \times \frac{+2}{100} \times 1.0 = +\text{£}2{,}000$$
$$\text{Adjusted revenue (W1)} = 100{,}000 - 2{,}000 + 2{,}000 = \text{£}100{,}000$$

All weeks (same fragment percentages):

Week1234
$\delta_{\text{FX}}$-£2,000-£2,100-£1,960-£2,200
$\delta_{\text{inf}}$+£2,000+£2,100+£1,960+£2,200
Adjusted revenue£100,000£105,000£98,000£110,000

In this case, the two fragments happen to cancel perfectly. Now suppose FX strengthens 5% but inflation is only +1%:

Week1234
$\delta_{\text{FX}}$-£2,000-£2,100-£1,960-£2,200
$\delta_{\text{inf}}$+£1,000+£1,050+£980+£1,100
Adjusted revenue£99,000£103,950£97,020£108,900

Balance cascade: net flow per week = adjusted revenue - costs.

Week1234
Net flow£19,000£21,950£18,020£23,900
Opening£200,000£219,000£240,950£258,970
Closing£219,000£240,950£258,970£282,870

Compare with sequential application (FX then inflation): sequential would compute $99{,}000 \times 1.01 = \text{£}99{,}990$ for week 1 (applying FX first, then inflation to the FX-adjusted value). Independent gives £99,000. The difference is £990 in week 1 alone. Over 260 weeks, these small differences accumulate and — more critically — they make it impossible to produce a clean attribution table.

With independent application, the attribution is unambiguous: FX reduced revenue by £2,400/week and inflation added £1,200/week, for a net impact of -£1,200/week. This decomposition is exact — the parts sum to the whole. With sequential application, the "inflation impact" depends on whether FX was applied first, making attribution inherently order-dependent and therefore arbitrary.

Historical note

Scenario planning was pioneered at Royal Dutch Shell in the early 1970s by Pierre Wack. Shell's scenarios famously anticipated the 1973 oil crisis, enabling the company to respond faster than competitors. Stress testing became a regulatory requirement after the 2008 financial crisis, formalised in the Basel II/III banking accords. Today, UK banks must submit annual stress test results to the Bank of England's Prudential Regulation Authority.

What breaks

Without independent application: sequential application introduces order dependence. Two analysts applying the same fragments in different orders get different results. Worse, each fragment's marginal impact becomes impossible to isolate — you cannot say "FX cost us exactly £X" because the FX delta depends on whether inflation was applied first.

Without the balance cascade: opening balances no longer match prior closing balances. The cash position in week 10 would still reflect the old week 9 closing, creating a discontinuity. The balance sheet would not balance, and the accounting identity $A = L + E$ would be violated.

7. Exercises

Exercise 1 (Academic)

Prove that independent fragment application is commutative for any number of fragments $n$.

Solution

Let fragments have deltas $\delta_1, \delta_2, \ldots, \delta_n$ where $\delta_i = B \cdot (p_i / 100) \cdot \iota_i$. Each $\delta_i$ depends only on the baseline $B$ (a constant) and the fragment's own parameters. The adjusted value is:

$$\text{adjusted} = B + \sum_{i=1}^{n} \delta_i$$

Let $\sigma$ be any permutation of $\{1, 2, \ldots, n\}$. Then:

$$B + \sum_{i=1}^{n} \delta_{\sigma(i)} = B + \sum_{i=1}^{n} \delta_i$$

because finite sums of real numbers are invariant under permutation (commutativity and associativity of addition). The result is identical for any ordering. ∎

Exercise 2 (Applied)

A company has baseline weekly revenue of £120,000 (60% domestic, 40% export), weekly costs of £95,000 (including £15,000 debt service and £40,000 payroll), and opening cash of £150,000. Apply FX -5% and inflation +2% independently over 4 weeks, then cascade the balances.

Solution

Revenue fragments (applied to £120,000 baseline each week):

$\delta_{\text{FX}} = 120{,}000 \times (-5/100) \times 0.40 = -\text{£}2{,}400$

$\delta_{\text{inf}} = 120{,}000 \times (2/100) \times 1.0 = +\text{£}2{,}400$

Adjusted revenue = £120,000 - £2,400 + £2,400 = £120,000 each week (fragments cancel).

If inflation were only +1%: $\delta_{\text{inf}} = +\text{£}1{,}200$. Adjusted revenue = £120,000 - £2,400 + £1,200 = £118,800.

Using the +1% inflation scenario for a more interesting cascade:

Net flow per week: £118,800 - £95,000 = £23,800.

Week1234
Opening£150,000£173,800£197,600£221,400
Net£23,800£23,800£23,800£23,800
Closing£173,800£197,600£221,400£245,200

Each week's opening equals the prior week's closing. The cascade is consistent.

Exercise 3 (Applied)

A company has £15,000 weekly debt service. The baseline interest rate is 5.25%. If the Bank of England raises rates to 5.75%, compute the weekly interest impact using the macro formula. Then apply it as a fragment alongside an employer NI increase from 13.8% to 15.0% on a £40,000 weekly payroll. Show the independent application.

Solution

Interest impact:

$$\text{impact} = 15{,}000 \times 0.70 \times \left(\frac{5.75}{5.25} - 1\right) = 10{,}500 \times 0.09524 = +\text{£}1{,}000/\text{week}$$

(Costs increase — the company pays more interest.)

NI impact:

$$\text{impact} = 40{,}000 \times 0.60 \times \frac{15.0 - 13.8}{100} = 24{,}000 \times 0.012 = +\text{£}288/\text{week}$$

Independent application on costs baseline of £95,000/week:

$\delta_{\text{interest}} = +\text{£}1{,}000$. $\delta_{\text{NI}} = +\text{£}288$. Adjusted costs = £95,000 + £1,000 + £288 = £96,288/week.

Each fragment's impact is separately attributable: "rate rise costs us £1,000/week; NI rise costs us £288/week."

Fragments operate independently from the baseline — never from each other — and every modified flow cascades forward through every subsequent balance.
