Course–Model Alignment

What’s actually in the forecasting engine, what’s loosely there, and what’s not there at all.

Every concept below was checked against the source code of the forecasting model. Three tiers: core (implemented and load-bearing), lightly present (used in one place or as a minor input), and not in the model (taught for background but absent from the codebase).

1. Core — directly implemented in the model

These concepts have dedicated functions, appear in multiple engine paths, or drive key outputs. You need all of them to rebuild the model.

  • Arithmetic Mean
    Base monthly revenue, seasonal index averaging, invoice counts. Used throughout.
  • Median
    DSO and DPO computation, recurring expense detection (median interval, median amount). The primary robust average.
  • Standard Deviation
    Component of CV. Computed via statistics.stdev in detector and quality modules.
  • Coefficient of Variation (CV)
    Recurring expense regularity test (reject if CV > 0.35). Uncertainty base rate calibration (CV thresholds at 0.3 and 0.6). Confidence scoring.
  • Percentiles (P25, P50, P75, P90)
    Invoice distribution characterisation in quality.py. Fed to assumptions tab and LLM context.
  • Recurring Expense Detection
    Groups bills by (supplier, category), computes median interval and CV, classifies frequency, assigns confidence. Full pipeline in detector.py.
  • Frequency Classification
    Maps median payment interval to weekly/monthly/quarterly/annual via defined bands. Used in detector.py.
  • Compound Interest / Continuous Compounding
    $(1+g)^{(w-1)/52}$ powers every projection — revenue growth, inflation, variable cost escalation. Both engines.
  • CAGR (Simplified)
    Revenue growth rate: (recent_12mo_avg / prior_12mo_avg) - 1, clamped to [-20%, +25%]. Computed in revenue.py.
  • Annuity Formula
    $PMT = Pr/(1-(1+r)^{-n})$. Used in event_injection.py :: _new_loan() for amortising loan payments.
  • Amortisation Schedule
    Interest/principal split each week: interest = remaining × rate, principal = PMT - interest. Built in _new_loan().
  • CVP Cost Classification
    Fixed/variable/semi-variable. Drives how each cost stream is projected. Category mapper + expense analyser + both engines.
  • Blended Growth Rate
    (inflation + growth) / 2 for the residual cost bucket. One line, but architecturally important. Both engines.
  • Depreciation Stripping
    Removing non-cash depreciation from P&L expenses to get cash-basis costs. Used in ingestion, PL engine, residual calculation.
  • Frequency Normalisation
    Weekly ↔ monthly conversion (4.33 factor). FREQUENCY_TO_MONTHLY, FREQUENCY_TO_WEEKLY in five-year engine.
  • Centred Moving Average (12-month)
    Half-weight endpoints. Hand-implemented in seasonality.py :: compute_seasonal_indices().
  • Ratio-to-Moving-Average Decomposition
    $SI(t) = x(t) / CMA(t)$, averaged by month, normalised to sum = 12. Core of seasonal extraction.
  • Seasonal Normalisation
    Rescale indices so $\sum = 12$. Guarantees no net annual inflation/deflation from seasonality.
  • Seasonal Index Floor with Redistribution
    Clamp ≥ 0.20, redistribute surplus iteratively. _apply_floor() in seasonality.py.
  • Dampened Seasonality
    $1 + (S-1)(1-f)$ where $f$ = fixed ratio. Applied to residual costs in both engines.
  • Accounting Identity ($A = L + E$)
    Enforced every week in the BS engine. Cash-as-plug is the mechanism. The constraint the whole model obeys.
  • P&L Projection (Revenue × Growth × Seasonality)
    The multiplicative revenue formula plus CVP cost projection. The core of pl_engine.py and five_year/engine.py.
  • Balance Sheet Roll-Forward
    Receivables, payables, tax accruals, PPE, retained earnings, cash plug. bs_engine.py :: roll_forward_bs().
  • Cash as Plug
    cash = total_L_E - non_cash_assets. Forces BS to balance. Both bs_engine.py and orchestrator.py.
  • VAT Accrual and Quarterly Drain
    Output - input VAT, accumulates weekly, drains on first week of each VAT quarter. bs_engine.py.
  • PAYE/NI Accrual and Monthly Drain
    staff_costs × 0.25, drains every 4th week. bs_engine.py.
  • Corporation Tax
    max(0, PBT × rate). Accumulates weekly, drains annually. pl_engine.py and bs_engine.py.
  • Employer NI and Pension
    max(0, salary - threshold) × ni_rate and salary × pension_pct. employer_cost_calculator.py and event injection.
  • Indirect Cash Flow Derivation
    $CFO = NP + dep - \Delta AR + \Delta AP + \Delta\text{tax}$. cf_engine.py :: derive_cf().
  • DSO and DPO
    Median payment delays. payment.py :: compute_dso(), compute_dpo(). Drives receivables/payables on BS.
  • Cash Conversion Cycle
    $CCC = DSO - DPO$. Computed in orchestrator.py :: compute_business_metrics().
  • DSO/DPO Trend Extrapolation
    effective_dso(w) = clamp(base + slope × w/13, 7, 120). bs_engine.py :: _effective_dso().
  • DSO Drag on Cash Receipts
    drag = max(0.70, 1 - annual_drag × year). Applied when trend > 1 day/quarter. Five-year engine.
  • Square Root of Time Uncertainty
    $\sigma(w) = \min(BASE + k\sqrt{w-1}, MAX)$. Both engines. Derived from Wiener process / random walk variance.
  • CV-Informed Base Uncertainty Rate
    CV < 0.3 → 3%, CV > 0.6 → 8%, else 5%. _select_base_rate() in uncertainty modules.
  • Fan Charts (Symmetric Spread)
    spread = |balance| × σ(w). Simplified Bank of England approach. Both engines.
  • Quadrature Combination
    $\sigma_{\text{total}} = \sqrt{\sigma_1^2 + \sigma_2^2}$. heuristic.py :: _combine_quadrature().
  • Z-Score Confidence Intervals
    1.645 / 1.96 / 2.576 for 90% / 95% / 99%. CONFIDENCE_Z_SCORES in heuristic.py.
  • Multiplier / Delta / Override Adjustments
    Three types, canonical order. adjustment_engine.py :: apply_scenario_adjustments().
  • DELAY Adjustment (Temporal Shift)
    Shift cash flows by offset weeks. Adjustment engine and five-year engine.
  • Independent Fragment Application
    Each fragment applies to original baseline, not running total. fragment_applicator.py.
  • Balance Cascade
    closing(w) = opening(w) + net(w). Recalculated after any modification. Three separate implementations.
  • FX / Interest Rate / Employer NI Macro Impacts
    Relative change × exposed amount. macro_fragment_generator.py.
  • Gross Margin
    1 - COGS/revenue. Derived in assumptions.py :: _derive_gross_margin().
  • Capex Rate
    min(0.15, depreciation/income). assumptions.py :: _derive_capex().
  • Dividend Payout Ratio
    min(1, dividends/(retained_profit + dividends)). assumptions.py :: _derive_dividend_pct().
  • Residual Cost Calculation
    max(0, (|opex| - dep) / months - COGS - named_opex). PL engine and five-year engine.
  • Three-Way Forecast (P&L → BS → CF)
    The integration layer. orchestrator.py :: compute_three_way_forecast().
  • Week-to-Month Aggregation
    Flows sum, stocks snapshot last week. aggregate_weeks_to_months() in both engines.
  • Straight-Line Depreciation
    cost / (life × 52) per week. Event injection for asset purchases.
  • Interest Expense
    balance × (base_rate + spread) / 52. pl_engine.py.
  • Direct Method Cash Flow (13-Week)
    Match invoices/bills to payment weeks by due date. DirectMethodForecastStrategy.
  • Expense-to-Revenue Ratio
    total_monthly_recurring / monthly_income. expense_analyser.py.
  • Customer Concentration
    Revenue share of top 1/3/5 customers. quality.py :: compute_customer_concentration().

2. Lightly present — used in one place or as a minor input

These are in the codebase but only just. One helper function, one line, or one edge-case guard.

  • OLS Linear Regression (Slope Only)
    Used in exactly one function: payment.py :: _linear_slope() for DSO/DPO trend detection. No intercept, no R², no residual analysis. Full page is overkill.
  • Variance
    Never computed directly. Only appears as an intermediate step to get standard deviation, which is then divided by the mean to get CV.
  • Growth Rate Clamping
    Clamp to [-20%, +25%]. Two lines of code in revenue.py. Philosophically important but mathematically trivial.
  • Inflation Compounding
    Same formula as revenue compounding but with the inflation rate. Applied to fixed costs. One line in each engine.
  • Fixed Ratio (Fixed Cost Proportion)
    total_fixed / total_monthly_recurring. Computed once in expense_analyser.py, fed to the dampening formula.
  • Driver Override Resolution
    Latest start_week wins when multiple overrides overlap. One function: driver_overrides.py :: get_driver_value().
  • Period Months Calculation
    days / 30.44. Two lines in pl_engine.py and expense_analyser.py.
  • Punctual Event BS Timing
    Mini-ledger for one-off invoices with DSO-based collection lag. punctual_bs.py. Niche edge case.

3. Not in the model — taught for background only

These concepts appear in the course but have no corresponding code in the forecasting engine. They are either pedagogical context, theoretical tangents, or prerequisites for understanding something that is in the model.

  • R² (Coefficient of Determination)
    Taught as a full section with worked example. Never computed anywhere in the codebase.
  • Geometric / Orthogonal Projection Interpretation of OLS
    Pure theory. References a “Phase 6 (Linear Algebra)” that doesn’t exist. Not in the codebase.
  • Tukey Fences / IQR Outlier Detection
    1.5 × IQR rule for outliers. Not implemented. The engine uses CV thresholds instead.
  • Five-Number Summary / Box Plots
    Not computed. The page claims it’s used “for every invoice stream” — this is incorrect.
  • Weighted Means
    The page claims “weighted means appear throughout the engine.” They do not. No weighted mean in the inventory.
  • Breakdown Point of Estimators
    Taught to justify why median is used for DSO. Valid pedagogy, but not in the code.
  • L1 Minimiser Property of the Median
    Formal proof that the median minimises absolute deviations. Not used in the engine.
  • Derivation of Euler’s Number $e$
    $\lim_{n \to \infty}(1+1/n)^n$. Historical background. The codebase uses (1+r)**frac without referencing $e$.
  • Bernoulli’s Inequality
    $(1+r)^n \geq 1+rn$. Proved to show compound beats simple. Not in the code.
  • L’Hôpital’s Rule (Zero-Rate Edge Case)
    Used to show the annuity formula degenerates to $P/n$ at zero interest. The code handles this as a simple if rate == 0 guard.
  • AM-GM Inequality Proof
    Proves arithmetic mean ≥ geometric mean to justify using AM for the blended rate. The code just computes (a+b)/2.
  • Low-Pass Filter / Frequency Response Framing
    The CMA explained as convolution with a rectangular kernel. The code is simple arithmetic — no signal processing.
  • ARIMA Model Structure
    Taught to explain why it was not used. Deliberately absent from the codebase.
  • Wiener Process Formal Definition
    Three axioms of $W(t)$. The code uses sqrt(w-1) — the consequence, not the formalism.
  • Fractional Brownian Motion / Hurst Parameter
    Mentioned as a generalisation. Not in the codebase.
  • Central Limit Theorem
    Invoked to justify the normal distribution assumption for z-scores. Not computed.
  • Normal Distribution Theory
    The z-score values are hardcoded constants. No probability density computation in the engine.
